lesson 01

How the Web Works

Before you build anything, you need to understand the system you're building in. This lesson demystifies the web — what happens when you type a URL, and where your code lives.

THE CORE IDEA

The web is just computers talking to each other. When you type google.com into a browser, your computer asks another computer (a server) "can I have the files for this page please?" That server sends back some files — HTML, CSS, JavaScript — and your browser turns those files into what you see on screen.

That's it. Every website you've ever visited is just files on a computer somewhere. Your job as a developer is to create those files and put them on a computer that other people can reach.

THE THREE FILES THAT MAKE EVERY WEBSITE

HTML — the structure. The bones. Text, headings, links, images.
CSS — the style. Colours, fonts, layout, spacing.
JavaScript — the behaviour. Clicks, animations, dynamic content.

Think of it as: HTML = the building, CSS = the interior design, JS = the electricity.

lesson 02

Your First HTML Page

HTML is the language of structure. In this lesson you write your first real HTML file, understand how tags work, and build the skeleton of your Phase 1 project.

HOW HTML WORKS

HTML uses tags to label content. A tag opens with <tagname> and closes with </tagname>. Everything between them is the content. Think of tags as instructions to the browser: "this is a heading", "this is a paragraph", "this is an image".

You don't need to memorise every tag. You need to understand the concept, know the common ones, and know how to look up the rest. That's how real developers work.

HTML · skeleton of every webpage
<!-- This is the basic structure of every HTML page -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <title>My First Page</title>
  </head>
  <body>

    <h1>Hello, I'm building things.</h1>
    <p>This is my first HTML page.</p>

    <!-- A link -->
    <a href="https://github.com">My GitHub</a>

  </body>
</html>

lesson 03

Styling with CSS

CSS transforms your plain HTML skeleton into something that looks intentional. This is where your design background becomes a superpower — you already have taste. Now you have the tools to express it.

HOW CSS WORKS

CSS works by selecting HTML elements and applying rules to them. A rule says: "find all elements matching this selector, and apply these property-value pairs."

The selector can be a tag name, a class (starting with .), or an ID (starting with #). Properties are things like color, font-size, background, padding. Values are what you set them to.

If you studied design, think of CSS as your style guide made executable. Every decision you'd make in a design brief — typography, colour, spacing, layout — CSS is how you implement it in code.

CSS · styling your personal site
/* CSS variables — define your palette once, use everywhere */
:root {
  --bg: #0a0a0a;
  --text: #f0f0f0;
  --accent: #c6f135;  /* your brand colour */
}

/* Reset default browser styles */
* { box-sizing: border-box; margin: 0; padding: 0; }

body {
  background: var(--bg);
  color: var(--text);
  font-family: 'Inter', sans-serif;
  line-height: 1.6;
}

h1 {
  font-size: 2.5rem;
  color: var(--accent);
  margin-bottom: 1rem;
}

/* Class selector — apply to any element with class="card" */
.card {
  background: #1a1a1a;
  padding: 1.5rem;
  border: 1px solid #333;
  border-radius: 4px;
}
THE LAYOUT MODEL — UNDERSTAND THIS ONCE, USE IT FOREVER

Every HTML element is a rectangular box. That box has: content (the text/image), padding (space inside the border), border (the edge), and margin (space outside). This is the CSS box model. Understanding it explains 80% of layout bugs.

For layout, Flexbox is what you'll use most at this stage. It lets you arrange items in a row or column, distribute space between them, and align them — with very clean, readable CSS.

CSS · flexbox for layout
/* Flexbox — your main layout tool */
.nav {
  display: flex;            /* turn it on */
  justify-content: space-between; /* spread items */
  align-items: center;    /* vertical centre */
  padding: 1rem 2rem;
  border-bottom: 1px solid #333;
}

.cards-row {
  display: flex;
  gap: 1rem;              /* space between cards */
  flex-wrap: wrap;         /* wrap to next line on mobile */
}

/* Make it mobile-friendly */
@media (max-width: 600px) {
  .nav { flex-direction: column; gap: 1rem; }
}
YOUR DESIGN BACKGROUND — USE IT HERE

Before you write CSS, sketch your layout on paper first (or in Canva, Figma free tier, whatever you have). Decide your colour palette — maximum 3 colours. Choose one font. Define spacing rules. This is your style guide. CSS is just how you implement it. Developers who think in systems build better UIs.

lesson 04

Python Basics + AI Pair Programming

Python is your second language — and your gateway to AI tools, automation, and backend work. In this lesson you write real Python, and you learn to use AI as a coding partner (not a crutch).

WHY PYTHON FIRST

Python reads almost like plain English, which makes it ideal for learning logic without fighting syntax. It's also the dominant language for AI/ML, data, automation, and scripting — the things you'll build in Phase 3 and 4. Learning Python now means every AI tool you touch later will feel familiar.

You don't need to become a Python expert in Phase 1. You need to understand: variables, data types, functions, loops, and conditionals. That's the foundation everything else builds on.

Python · the fundamentals
# Variables — storing values
name = "Tina"
age = 17
is_freelancer = True

# f-strings — embed variables in text
print(f"Hi, I'm {name} and I'm {age}.")

# Functions — reusable blocks of logic
def greet(person_name):
    return f"Hey {person_name}, welcome to StackUp."

# Calling the function
print(greet("Jordan"))

# Lists — a collection of values
skills = ["HTML", "CSS", "Python", "Git"]

# Loops — do something for each item
for skill in skills:
    print(f"I'm learning: {skill}")

# Conditionals — make decisions
if age >= 16:
    print("Old enough to freelance.")
else:
    print("Keep learning — you're almost there.")
AI PAIR PROGRAMMING — THE RIGHT WAY

Wrong way: "Write me a Python script that does X." Copy, paste, done.
Right way: Write the structure yourself first. Get stuck. Then: "I'm writing a Python function that takes a name and returns a greeting. Here's what I have so far: [code]. I'm getting this error: [error]. What's wrong?"

The difference is understanding. The wrong way makes you dependent. The right way makes you dangerous.

Python · starter scaffold for the activity
# rate_calculator.py — your first useful script

def calculate_quote(hourly_rate, estimated_hours, revision_buffer):
    """
    Calculate a project quote including revision buffer.
    revision_buffer: 0.2 means add 20% for revisions
    """
    base = hourly_rate * estimated_hours
    total = base * (1 + revision_buffer)
    return round(total, 2)

def aud_to_usd(aud_amount, rate=0.65):
    return round(aud_amount * rate, 2)

# --- Run it ---
quote_aud = calculate_quote(
    hourly_rate=50,
    estimated_hours=10,
    revision_buffer=0.2
)

print(f"Project quote (AUD): ${quote_aud}")
print(f"Project quote (USD): ${aud_to_usd(quote_aud)}")

lesson 05

Git — Version Control from Day One

Git is non-negotiable. It's how every developer in the world saves and shares their work. In this lesson you create your GitHub profile, make your first commit, and publish your personal site live.

WHAT GIT ACTUALLY IS

Git is a tool that tracks every change you make to your code. Every time you commit, Git saves a snapshot of your project. If you break something, you can go back. If you're working with others, you can merge your changes. Every professional developer uses it — always, on every project.

GitHub is a website that stores your Git repositories online. Your GitHub profile is your developer portfolio. Every commit you make is visible. A GitHub profile with real commits is worth more than a CV in your first job search.

THE FOUR COMMANDS YOU'LL USE 90% OF THE TIME

git init — start tracking a folder
git add . — stage all changed files
git commit -m "message" — save a snapshot
git push — send to GitHub

Terminal · your first repo workflow
# 1. Initialise Git in your project folder
git init

# 2. Tell Git who you are (one-time setup)
git config --global user.email "you@email.com"
git config --global user.name "Your Name"

# 3. Stage everything in the folder
git add .

# 4. Commit with a meaningful message
git commit -m "Initial commit: personal site skeleton"

# 5. Connect to GitHub (create repo on github.com first)
git remote add origin https://github.com/yourusername/my-site.git

# 6. Push to GitHub
git push -u origin main

# After this, your daily workflow is just:
git add .
git commit -m "describe what you changed"
git push
DEPLOY TO GITHUB PAGES — YOUR SITE GOES LIVE
01

Push your my-site folder to GitHub using the commands above. Create a new repo on github.com called my-site first.

02

On GitHub, go to your repo → Settings → Pages → Source: Deploy from branch → main → Save.

03

Wait 2 minutes. Refresh. You'll see a URL: yourusername.github.io/my-site. That's your live site.

04

Make a change to your HTML. Save it. Run git add . && git commit -m "update" && git push. Your live site updates automatically.

COMMIT MESSAGE BEST PRACTICE

Good: "Add navigation bar with 3 links"
Good: "Fix mobile layout on hero section"
Bad: "update" / "fix" / "asdfgh"

Write commit messages as if a colleague needs to understand what you changed in 6 months. This is also how clients and employers read your GitHub.

LOGIC IN CODE — BOOLEAN THINKING

Logic in programming is binary: something is either True or False. Conditions, comparisons, and decisions all reduce to this. Understanding boolean logic means you can reason about any conditional, any filter, any access control — from a simple if-statement to a database query.

Python · logic and decision making
# Boolean logic — True or False is all there is
user_logged_in = True
has_paid = False

# and: both must be True
if user_logged_in and has_paid:
    print("Access granted")
else:
    print("Please purchase to continue")

# or: at least one must be True
is_admin = False
if has_paid or is_admin:
    print("You can access this content")

# not: flip it
if not has_paid:
    print("Reminder: payment required")

# Comparison operators
score = 85
if score >= 90:
    grade = "A"
elif score >= 75:
    grade = "B"
else:
    grade = "C"

print(f"Grade: {grade}")
